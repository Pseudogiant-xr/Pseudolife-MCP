# Pseudolife-MCP

[英語版 README（正典）](../../README.md) — 同期バージョン: v3 (2026-07-19)

**Claude Code、Codex、その他の MCP クライアントに永続的な長期メモリを提供します。**

コーディングエージェントに、セッションをまたいで保持される長期メモリを与える MCP サーバーです。コンテキストの圧縮や新しいタスクの開始を経ても記憶は失われません。コーディングエージェントが知性であり、このサーバーはそのディスク上のメモリという位置づけです。

提供する機能:

- **記憶らしく熟成する連想メモリ** — 類似度でランク付けされたメモリバンドによる「新しさの連続体」に、矛盾検出と置き換え（supersession）を組み合わせています。訂正は古い回答の横に積み上がるのではなく、それを置き換えます。
- **雰囲気ではなく正準的なファクト** — `entity.attribute` スロットごとに *現在の* 値は 1 つだけ。訂正は黙って上書きするのではなく明示的に置き換え、完全なバージョン履歴が保持されます。
- **ドリーム（Dreams）** — ユーザーが離席している間に、抽出器がメモリストリームを正準的なファクトとナレッジグラフへ統合します。
- **自身の作業から得る教訓** — 成功、行き詰まり、そしてユーザーからの訂正が「やるべきこと・避けるべきこと」のガイダンスとなり、毎回のセッション開始時に提示されます。
- **思考の様子を観察できる Web コンソール** — Cortex Console では、メモリストリーム、ファクト履歴、ナレッジグラフのアトラス、セッションのエピソード、ドキュメント RAG を閲覧できます。

## クイックスタート

Docker と、Claude Code・Codex のいずれか（または両方）が必要です。クローンから最初の記憶までコマンド 1 つで到達できます（デフォルトのクライアントは Claude です）:

```bash
git clone https://github.com/Pseudogiant-xr/Pseudolife-MCP.git
cd Pseudolife-MCP
ops/install.sh          # Linux / macOS
ops\install.ps1         # Windows (pwsh 7+)
# Codex: add --client codex / -Client codex
# Both:  add --client both  / -Client both
```

インストーラーは前提条件をチェックし（不足があれば、そのまま実行できる修正コマンドを 1 行で表示します）、どのドリーム抽出器を使うかを尋ねます — Max プラン経由の Claude Sonnet（最も軽量なインストール）か、プラン不要で動作する同梱のローカルモデルかを選べます。その後スタックを起動し、選択したクライアントの接続を設定し（セッション開始時のブリーフィングフック、`~/.claude/CLAUDE.md` または `~/.codex/AGENTS.md` 内の常設メモリループ指示、MCP サーバーの登録）、デーモンのヘルスチェックまで行います。冪等なので、いつでも再実行できます。

デーモンが起動していれば、Claude Code の**プラグイン**が、セッション開始時のメモリブリーフィングに加えて `/dream` と `/memory-status` コマンドを追加します — MCP サーバー自体はインストーラーが登録するため、プラグインがツールを二重に登録することはありません:

```
/plugin marketplace add Pseudogiant-xr/Pseudolife-MCP
/plugin install pseudolife-memory@pseudolife-mcp
```

Codex はサーバーを直接登録します:

```bash
codex mcp add pseudolife-memory --url http://127.0.0.1:8765/mcp
```

あとは、どちらのコーディングエージェントでも、*「ステージングのマシンは haze-02 だと覚えておいて」* と伝えるだけです。数日後の新しいセッションで *「ステージングはどのマシン？」* と尋ねれば、メモリから答えが返ってきます。すべての内容は `http://127.0.0.1:8765/ui/` の Cortex Console で閲覧できます。

## 仕組み

エージェントは作業しながら主張を 1 件ずつ保存します（`memory_store`、`memory_fact_set`）。サプライズゲートがほぼ重複する内容を弾きます。セッションの合間には、**ドリーム**がストリームを正準的なファクト、グラフのリレーション、手続き的な教訓へと蒸留します。毎回のセッション開始時には、ブリーフィングが「メモリが確信を持てていない点」「過去の作業から得た教訓」「前回の作業の続き」を注入します。検索はメモリバンドに対するセマンティック検索と正準ファクトストアを組み合わせるため、訂正済みの回答が古い回答に勝ちます。
