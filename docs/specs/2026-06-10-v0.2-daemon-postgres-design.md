# Pseudolife-MCP v0.2 — Memory daemon on Postgres (design)

**Date:** 2026-06-10
**Status:** Approved by user (pending spec review)
**Drivers:** Fable 5 review of 2026-06-10 + user direction (Postgres, daemonize, full knowledge graph)

---

## 1. Context and goals

A full review of the v0.1 system (2026-06-10) found four glaring issues and a set of
smaller ones:

1. **Silent-wipe hazard.** `torch.save` writes `cms_state.pt` / `cortex_state.pt`
   in place with no atomic rename and no backup. `CMS.load` swallows corrupt-file
   errors and starts empty; the 30s autosave then overwrites the corrupt file.
   A crash mid-write can permanently destroy the whole bank.
2. **Concurrent-session clobbering.** Every Claude Code session spawns its own
   stdio server process sharing one data dir. Autosave is fingerprint-gated
   (idle processes don't clobber) but two *active* writers are last-writer-wins:
   one session's stores are lost when another saves later.
3. **Advertised auto-promotion doesn't exist.** `memory_store` docs promise
   "X is Y" / "my X" cortex auto-capture; `slots.py` only implements chat-era
   patterns (`named X`, `I have a <breed> cat`); `_MY_NOUN_RE` is dead code.
   Verified live: 0 auto-promoted facts from 24 real stores.
4. **Meta-filter collides with this project's own domain.** Patterns like
   `surprise (score|gating|threshold)` reject deliberate stores about the
   memory system itself.

Smaller: neural blend weight (0.6) rides an undertrained MLP at cold start;
recency half-life (1h base) is chat-shaped, not session-shaped; README test
count stale; `torch.load` is unhardened pickle.

**User-ratified direction:** rather than patching the `.pt` files, move to the
architecture the system should have long-term —

- a **single long-lived memory daemon** (MCP over HTTP) that all Claude Code
  sessions and any LAN agent box connect to;
- **Postgres 16 + pgvector + Apache AGE** (via Docker Desktop) as the storage
  engine;
- the cortex grown into a **real knowledge graph** (typed entities, typed
  temporal relations, aliases, traversal), with graph algorithms via NetworkX
  and optional openCypher via AGE.

## 2. Decision log

| Decision | Choice | Rationale / rejected alternatives |
|---|---|---|
| Process model | **Single daemon, MCP streamable-HTTP** | One writer kills the concurrency problem structurally; one torch instance (~600MB) instead of N; one warmup; LAN-shareable. Rejected: stdio-per-session + WAL/merge (hand-rolled coherence, N× RAM). |
| Storage engine | **Postgres 16 + pgvector + AGE** | User chose maximum power + future-proofing; enables direct SQL access by other tools; network-native; 25+ yr longevity. Rejected: SQLite (right-sized but user wants the stronger endgame; remains the documented downgrade path), Kùzu/Cozo (abandonment risk), LanceDB/SurrealDB (young), DuckDB (single-writer). |
| PG substrate | **Docker Desktop** (compose file in repo) | Only clean path to AGE on a Windows machine; pinned image; volume-backed. Rejected: native Windows PG (AGE build pain), WSL2-native (user upkeep). |
| AGE role | **Optional power layer, never a hard dependency** | AGE is Linux-first, lags PG majors (we pin PG 16). Graph core works on plain tables + recursive CTEs + NetworkX; AGE adds `memory_graph_query` (openCypher) when present; absence degrades gracefully. |
| Retrieval architecture | **Write-through in-memory cache** | Daemon keeps today's in-memory band structures + retrieval code (torch cosine, MLP blend, slot channel, BM25) unchanged; Postgres is durable truth underneath. Minimizes behavioral risk; the 207-test suite's semantics carry over. pgvector columns + HNSW indexes exist for external SQL consumers, not for the daemon's own hot path. |
| Band MLP weights | **Stay on disk (`weights.pt`), atomic** | They are a continuously-retrained cache, not source of truth. tmp + `os.replace`, one `.bak` rotation, `weights_only=True` load attempt with legacy fallback. Corrupt weights now cost retraining only. |
| Graph algorithms | **NetworkX in the daemon** | Pure-Python, 20-yr-old dependency; full algorithm library (paths, neighborhoods, centrality) at our scale (≤ tens of thousands of nodes). Rejected as core: AGE/Cypher (optional layer), RDF/SPARQL/OWL (formalism with no query pattern behind it — YAGNI). |
| Fallback transport | **stdio shim that finds/starts the daemon** | The shim never embeds storage — exactly one storage path exists. If the daemon is unreachable it attempts a detached spawn, retries, then errors clearly. |
| Weak-model formalism | **Ontology-lite (§5.3): closed relation registry + soft type hints + transitive/inverse inference computed on read** | A weak-model client (e.g. a ~27B-class model) drifts without constraints; the fix is formalism in the *substrate* (closed vocabulary, server-side inference), never in the *interface* (no query languages for weak models). Rejected: SPARQL/Cypher exposure to weak models (query composition adds degrees of freedom), OWL/DL reasoners (complexity nobody feeds), stored materialization of derived edges (invalidation burden; the graph is small enough to derive on read). |

## 3. Architecture

```
Claude Code session A ──HTTP──┐
Claude Code session B ──HTTP──┤
LAN agent box    ──HTTP+token─┼──► pseudolife-mcp daemon (single process)
  stdio shim (fallback) ──────┘        │ torch embedder + MIRAS bands (in-memory,
                                       │   write-through; semantics unchanged)
                                       ├──► Postgres 16 (Docker): entries, facts,
                                       │      entities, edges, episodes, meta
                                       │      (+ optional AGE graph mirror)
                                       └──► weights.pt (atomic; disposable cache)
```

### 3.1 Daemon

- `pseudolife-mcp serve` — runs FastMCP with `transport="streamable-http"`.
- Bind `127.0.0.1:8765` by default. Env:
  - `PSEUDOLIFE_MCP_HOST` / `PSEUDOLIFE_MCP_PORT`
  - `PSEUDOLIFE_MCP_TOKEN` — when set, requests must carry
    `Authorization: Bearer <token>`; REQUIRED when host ≠ 127.0.0.1
    (daemon refuses to bind non-loopback without it).
  - `PSEUDOLIFE_MCP_DATABASE_URL` — default
    `postgresql://pseudolife:pseudolife@127.0.0.1:5433/pseudolife_memory`.
  - `PSEUDOLIFE_MCP_DATA_DIR` — weights.pt location + legacy-migration source.
- Request handling stays serialized through the existing coarse service lock
  (correctness first; the workload is a few calls/minute).
- Postgres itself is never exposed to the LAN; only the daemon is.

### 3.2 stdio shim

`pseudolife-mcp` (no args, current entry point) becomes the shim:

1. Probe the daemon URL (`GET /health`, ~250ms timeout).
2. If absent: spawn `pseudolife-mcp serve` detached, poll health up to ~10s.
3. Proxy MCP stdio ⇄ HTTP for this session (the `mcp` SDK client does the
   bridging).
4. If the daemon can't be reached or started: return a structured error on
   every tool call naming the failure and the fix (`docker compose up -d`,
   `pseudolife-mcp serve`). It does NOT fall back to embedded storage.

### 3.3 Lifecycle on the host

- `ops/docker-compose.yml` — pinned image with PG 16 + pgvector + AGE
  (built from `ops/Dockerfile.pg` if no suitable upstream image is pinned),
  `restart: unless-stopped`, named volume, host port 5433 (avoids native-PG
  collisions).
- `ops/install-autostart.ps1` — registers a Task Scheduler logon task for
  `pseudolife-mcp serve`.
- `ops/backup.ps1` — `pg_dump` to `data/backups/` with simple rotation.

## 4. Storage layer (schema v8)

New module `pseudolife_memory/storage/postgres.py` behind a narrow seam
(`StorageBackend` protocol): `load_all()`, `insert_entry`, `update_entry`
(supersession/access/band moves), `delete_entries`, episode CRUD, fact CRUD,
graph CRUD, `meta` get/set. `MemoryService` and `CMS` call the seam only.

```sql
CREATE EXTENSION IF NOT EXISTS vector;          -- pgvector
-- AGE optional: CREATE EXTENSION age; detected at startup

CREATE TABLE meta (key TEXT PRIMARY KEY, value JSONB NOT NULL);

CREATE TABLE episodes (
  id TEXT PRIMARY KEY, title TEXT NOT NULL, hint TEXT,
  started_at DOUBLE PRECISION NOT NULL, ended_at DOUBLE PRECISION,
  closed_by_new_start BOOLEAN NOT NULL DEFAULT FALSE);

CREATE TABLE entries (
  id BIGSERIAL PRIMARY KEY,
  band TEXT NOT NULL,
  text TEXT NOT NULL,
  embedding vector(384) NOT NULL,
  surprise REAL NOT NULL DEFAULT 0,
  ts DOUBLE PRECISION NOT NULL,
  access_count INTEGER NOT NULL DEFAULT 0,
  source TEXT NOT NULL DEFAULT '',
  superseded_at DOUBLE PRECISION,
  superseded_by_text TEXT,
  last_logical_turn INTEGER,
  episode_id TEXT REFERENCES episodes(id),
  episode_title TEXT,
  tags JSONB NOT NULL DEFAULT '[]',
  slots JSONB NOT NULL DEFAULT '[]');
CREATE INDEX ON entries (band);
CREATE INDEX ON entries (ts);
CREATE INDEX ON entries (source);
CREATE INDEX ON entries USING hnsw (embedding vector_cosine_ops);

CREATE TABLE entities (
  id BIGSERIAL PRIMARY KEY,
  canonical TEXT NOT NULL UNIQUE,     -- normalized (lower, separators folded)
  display TEXT NOT NULL,
  etype TEXT,                         -- optional type tag ("project", "host", …)
  created_at DOUBLE PRECISION NOT NULL);

CREATE TABLE entity_aliases (
  alias TEXT PRIMARY KEY,             -- normalized
  entity_id BIGINT NOT NULL REFERENCES entities(id) ON DELETE CASCADE);

CREATE TABLE facts (                  -- the cortex
  id BIGSERIAL PRIMARY KEY,
  entity TEXT NOT NULL, attribute TEXT NOT NULL,      -- display forms
  entity_norm TEXT NOT NULL, attribute_norm TEXT NOT NULL,
  value TEXT NOT NULL, polarity TEXT NOT NULL DEFAULT '+',
  status TEXT NOT NULL,               -- current|superseded|contender|retired
  confidence REAL NOT NULL, origin TEXT,
  support JSONB NOT NULL DEFAULT '[]', provenance JSONB NOT NULL DEFAULT '[]',
  asserted_at DOUBLE PRECISION NOT NULL, last_confirmed DOUBLE PRECISION NOT NULL,
  supersedes_value TEXT, superseded_by_value TEXT, superseded_at DOUBLE PRECISION,
  embedding vector(384),
  entity_id BIGINT REFERENCES entities(id),           -- subject node
  object_entity_id BIGINT REFERENCES entities(id));   -- set when value IS an entity
CREATE INDEX ON facts (entity_norm, attribute_norm, status);

CREATE TABLE relations (              -- closed vocabulary (ontology-lite, §5.3)
  name TEXT PRIMARY KEY,              -- normalized ("depends-on")
  description TEXT NOT NULL,
  src_type TEXT, dst_type TEXT,       -- soft domain/range hints (warn, never reject)
  transitive BOOLEAN NOT NULL DEFAULT FALSE,
  inverse_of TEXT REFERENCES relations(name),
  builtin BOOLEAN NOT NULL DEFAULT FALSE,
  created_at DOUBLE PRECISION NOT NULL);

CREATE TABLE edges (                  -- typed entity↔entity relations
  id BIGSERIAL PRIMARY KEY,
  src_id BIGINT NOT NULL REFERENCES entities(id) ON DELETE CASCADE,
  relation TEXT NOT NULL REFERENCES relations(name),
  dst_id BIGINT NOT NULL REFERENCES entities(id) ON DELETE CASCADE,
  confidence REAL NOT NULL DEFAULT 0.8, origin TEXT,
  asserted_at DOUBLE PRECISION NOT NULL, superseded_at DOUBLE PRECISION,
  UNIQUE (src_id, relation, dst_id));
```

Write-through semantics: `CMS.store` → `INSERT` + append to in-memory band;
promotion → `UPDATE entries SET band=…`; supersede/delete → `UPDATE`/`DELETE`;
counters/tier-hits/surprise-history → `meta` (flushed with the debounced
autosave cadence — they are telemetry, not facts). Startup loads everything
into the existing in-memory structures. Embeddings round-trip through
pgvector; torch remains the hot-path scorer.

**Migration:** `initdb` creates schema; on daemon start with empty `entries`
and a v≤7 `cms_state.pt`/`cortex_state.pt` present in
`PSEUDOLIFE_MCP_DATA_DIR`, import everything in one transaction and rename the
files `*.pre-v8.bak`. Idempotent; never deletes originals.

**Weights:** `memory_state/weights.pt` holds per-band module/optimizer/EMA +
the new `update_count`. Atomic save (tmp + `os.replace`), single `.bak`
rotation, `torch.load(weights_only=True)` first with legacy fallback. Load
failure → log, start weights fresh, set `weights_reset` flag surfaced in
`memory_stats`.

## 5. Graph layer

### 5.1 Core (always on)

- **Auto-linking:** on any fact write, if `value` (normalized) matches an
  entity canonical/alias, set `object_entity_id`. On `memory_fact_set` /
  promotion, ensure the subject entity row exists.
- **New tools:**
  - `memory_graph_relate(src, relation, dst, origin?, confidence?)` — upsert a
    typed edge; entities auto-created; re-assertion bumps confidence;
    provenance tiers identical to facts; `relation` validated against the
    registry with fuzzy suggestions on miss (§5.3). Asserting a conflicting
    *functional* relation is out of scope v0.2 (edges are additive; remove
    with `memory_graph_unrelate`).
  - `memory_graph_unrelate(src, relation, dst)` — mark edge superseded.
  - `memory_alias(entity, alias)` — bind alias → entity; all fact/graph
    lookups resolve aliases first.
  - `memory_graph(entity, depth=1, include_facts=true)` — subgraph within
    `depth` hops (cap 3): nodes, edges, each node's current facts, plus
    `paths` when exactly two entities are named (`entity="a..b"` syntax is NOT
    used; a `to=` param handles paths). Served by recursive CTE → NetworkX
    digraph → compact JSON.
- `memory_facts` / `memory_fact_get` answers gain `entity_id` + alias info.

### 5.2 AGE layer (optional)

- Startup: `CREATE EXTENSION age` probe; if present, ensure graph
  `pseudolife` exists and lazily mirror `entities`/`edges` (idempotent
  upserts on write; full re-sync command `pseudolife-mcp age-sync`).
- Tool `memory_graph_query(cypher, limit=50)` — read-only openCypher via AGE
  (`SELECT * FROM cypher('pseudolife', $$ … $$)`); rejects mutating clauses
  (CREATE/SET/DELETE/MERGE) by keyword screen; absent AGE → structured
  "age_unavailable" error naming the compose image as the fix.
- **Strong-model tool.** Cypher composition is a degrees-of-freedom hazard
  for weak clients; weak-model deployments must not expose this tool (§5.3).

### 5.3 Ontology-lite — weak-model guardrails

Motivation: the same bank can serve both a strong client (Claude) and a
weak-model agent (e.g. a ~27B-class model). Weak models drift: they invent relation-name variants
(`depends_on` / `dependsOn` / `uses`), fragment the graph, and cannot reliably
chain multi-hop conclusions. The remedy is formalism in the **substrate** —
closed vocabularies, server-side inference — never in the **interface**
(query languages). This section is the deliberate, scoped replacement for the
rejected RDF/OWL/SPARQL stack.

1. **Closed relation vocabulary.** `edges.relation` is a foreign key into the
   `relations` registry. Seed builtins:
   `depends-on` (transitive), `part-of` (transitive), `runs-on`
   (inverse `hosts`), `hosts`, `uses`, `configures`, `stores-data-in`,
   `related-to` (untyped catch-all). New relations are added only via
   `memory_relation_define(name, description, transitive?, inverse_of?,
   src_type?, dst_type?)` — a deliberate, strong-model act.
2. **Validation with suggestion, not silent failure.** A
   `memory_graph_relate` call with an unknown relation is rejected with a
   structured error carrying the top-3 fuzzy matches from the registry
   ("unknown relation 'dependsOn' — did you mean 'depends-on'?") and a hint
   that `related-to` is the lawful fallback. Deterministic normalization
   (lowercase, separators → `-`) runs before matching, so the common
   weak-model variants resolve without an error at all.
3. **Soft domain/range hints.** `relations.src_type` / `dst_type` are
   compared against `entities.etype` when both sides are typed. Mismatches
   **warn** (a `warnings: [...]` field on the response; the edge stores
   anyway) — a hard reject would put a weak model into retry loops or
   silent abandonment; a stored-with-warning edge keeps the bank growing
   while flagging the discrepancy, consistent with the contender philosophy.
4. **Inference computed on read.** Relations flagged `transitive` get their
   closure, and `inverse_of` pairs get mirrored edges, computed at query
   time by NetworkX inside `memory_graph` (the graph is small; no stored
   materialization, no invalidation problem). Derived edges return marked
   `derived: true` with rule provenance (`via: ["transitive:depends-on"]`),
   so a weak model receives complete multi-hop conclusions as plain facts —
   the server does the reasoning, the model does the reading.
5. **Per-client tool curation (weak-model deployment note).** A weak-model
   plugin should expose only the narrow, parameterized surface: `memory_search`,
   `memory_store`, `memory_fact_get` / `memory_fact_set`, `memory_graph`
   (read), `memory_graph_relate`. It must NOT expose `memory_graph_query`
   (Cypher), `memory_relation_define`, `memory_delete`, or
   `memory_fact_forget`. The prefetch-cortex-first pattern extends to
   prefetch-subgraph: the weak-model runtime calls `memory_graph` on the
   entities in play and injects the typed fact sheet before generation.

## 6. Phase 0 quick fixes (independent of storage)

1. **Meta-filter:** `memory.meta_filter.enabled` config (library default
   `true`); `_apply_mcp_defaults` sets `false`. Remove the two
   domain-colliding patterns (`surprise (score|gating|threshold)`,
   `memory bank…stores`) from the list entirely.
2. **Neural warmup ramp:** per-band persisted `update_count`; effective
   neural weight `= memory.neural_blend_weight × min(1, update_count /
   memory.neural_warmup_updates)`; defaults 0.6 / 50; `warmup_updates=0`
   restores exact v0.1 behavior. Exact-cosine weight is
   `1 − effective_neural_weight`.
3. **Dev-fact extractor** (`slots.py`): delete `_MY_NOUN_RE`; add
   conservative, sentence-bounded patterns —
   - `my <attr> is <value>` → `(user, attr, value)`
   - `<entity>'s <attr> is <value>` and `the <attr> of <entity> is <value>`
   - `<entity …> <attr-lexicon> is <value>` where the attribute is the
     longest lexicon match ending the subject (lexicon: default, port,
     version, timeout, host, address, ip, path, dir, directory, branch,
     language, status, name, url, endpoint, model, device, scope, threshold,
     capacity, schema, license, framework, editor, os, database);
   - single-line `key: value` / `key = value` with key ≤ 4 words, value
     non-empty, not inside code fences.
   - Guards: skip questions (`?`), skip negated copulas (`is not`/`isn't`),
     value length caps, no extraction from texts > 600 chars per sentence.
4. **Recency:** `memory.recency_base_half_life_s` (library default 3600);
   MCP default 86400 (ramp becomes 1d/2d/4d/…).
5. **Docs:** README persistence/tool/deploy sections rewritten for v0.2;
   test count corrected; security note (weights file is trusted-local pickle;
   DB carries the facts).

## 7. Testing

- **Logic suites (no PG):** extractor precision/recall (incl. false-positive
  guards on questions, negations, code), ramp math, meta-filter knob,
  recency config, restatement dedup — extend the existing pure suites.
- **Storage suite:** fixture resolves a PG via `PSEUDOLIFE_TEST_DATABASE_URL`
  or testcontainers (session-scoped container, per-test schema). Covers CRUD
  round-trips, migration from a synthesized v7 `.pt` pair, write-through
  consistency (in-memory view == SQL view after each op), weights atomicity +
  corrupt-file recovery.
- **Daemon suite:** start daemon against test PG; two concurrent HTTP clients
  interleave stores/searches — all stores visible, none lost; token auth
  enforced on non-loopback; shim spawn path smoke-tested.
- **Graph suite:** relate/alias/traverse with cycles, depth caps, path
  queries; ontology-lite — unknown-relation rejection with suggestions,
  normalization resolving `depends_on`→`depends-on`, soft type warnings
  stored-with-warning, transitive closure + inverse derivation marked
  `derived: true`; AGE tests marked skip-if-unavailable.
- Existing 207 tests stay green at every phase boundary (Phase 1 adapts their
  service construction to the test-PG fixture; assertions unchanged).

## 9. Risks and mitigations

- **AGE platform risk** → optional layer; PG pinned to 16; graph core never
  depends on it.
- **Docker Desktop not running** → compose `restart: unless-stopped`; shim
  and daemon errors name the exact recovery command; health endpoint.
- **Daemon down ⇒ no memory** → shim auto-start; Task Scheduler logon task;
  clear structured errors rather than silent absence.
- **Behavior drift in retrieval** → write-through cache preserves v0.1
  scoring code verbatim; ramp is config-gated with a v0.1-exact setting.
- **Migration bugs** → migration is read-only on sources (rename, never
  delete); single transaction; round-trip test from synthesized v7 state.
- **LAN exposure** → token mandatory off-loopback; PG never exposed.

## 10. Out of scope (deliberate)

- RDF/OWL/SPARQL and DL reasoners. Their weak-model-relevant benefits —
  closed vocabulary, soft typing, transitive/inverse inference — are
  delivered by ontology-lite (§5.3) without the formalism stack.
- Functional-relation conflict logic on edges (edges are additive in v0.2).
- Replacing in-process BM25 with `tsvector` ranking (noted as future work).
- Multi-daemon / cross-machine replication (Postgres makes it possible later).
- MCP sampling-based reflection (still blocked upstream).
